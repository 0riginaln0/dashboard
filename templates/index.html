This website demonstrates three different ways to refresh the content above:

    /
    Traditional full page reload — The current page's method: the entire page is reloaded from the server each time you navigate or refresh.
    Uses Jinja2 templating to render the HTML on the server.
    /sse
    Server-Sent Events — A persistent connection pushes updates from the server as soon as new data is available. Updates are broadcast to all connected clients simultaneously, so everyone sees the same changes in real time.
    Uses Fixi & Idiomorph to morph updated content into the page.
    /poll
    Polling — The browser asks the server for updates every few seconds. + a button to ask for the immediate server update
    Uses Fixi & Idiomorph & hyperscript to manage polling, DOM updates and dynamic button behaviour.

The backend is written in MicroPie, an «async Bottle».
